Services

Android development, database-backed systems, and Play Store publishing — end to end, or for a single stage of a project already in progress.

Android App Development

Native Android applications built in Kotlin or Java using Android Studio. This covers the full build: UI implementation, local storage (Room/SQLite), background services where needed (such as a foreground media-playback service), and testing across multiple Android API levels rather than a single device. Recent examples include a native music player with playlist management, an equalizer, and a sleep timer, built to target current API levels.

What's included

  • Requirements clarified into a written feature list before coding starts
  • UI built to Material Design conventions unless a custom design is provided
  • Testing on at least one physical device, not emulator-only
  • A working APK/AAB handed over, not just source files

Game Development

Casual and puzzle-style games with straightforward mechanics: level progression, scoring, and daily-challenge style content. Games are built with monetization in mind from the start — ad placements that don't interrupt core gameplay, and a difficulty curve that keeps players returning rather than dropping off after level one.

Backend & Database Systems

For apps or academic projects that need real data rather than local-only storage, I build backend APIs in Node.js/Express connected to Oracle Database or MySQL. This includes schema design, stored procedures and functions, views, triggers, and role-based authentication (for example, separate admin, agent, and customer roles in an insurance-management system). Past projects in this category include a full expense-tracking system and an insurance management platform with six modules covering agents, customers, policies, and claims.

App Maintenance & Compatibility Updates

Existing Android apps need upkeep as new Android versions ship. This service covers fixing compile errors introduced by SDK or dependency updates, patching bugs, and adding features to an app that's already live — without a full rewrite.

AdMob Monetization

Correct AdMob integration: setting up real ad units (not accidentally shipping Google's test ad unit ID to a production build, which is a mistake that can stall AdMob approval entirely), choosing ad formats and placements that follow Google's policies, and testing that ads load correctly before release.

Google Play Store Publishing

Handling the submission process itself: store listing copy, screenshots, content rating questionnaire, privacy policy requirements, and responding to any policy issues Google raises during review. This is often the stage where projects stall, so it's offered as a standalone service for apps built elsewhere that just need to get published correctly.

How a typical project runs

  1. Requirements discussion and written scope
  2. Development in stages, with progress shared along the way
  3. Testing on real devices across relevant Android versions
  4. Handover: source code, build files, and (if applicable) Play Store submission

For pricing or timelines specific to your project, see the contact page.
